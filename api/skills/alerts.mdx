---
title: "Alerts API"
description: "Create, list, remove, and check price and technical alerts with optional webhook callbacks."
---

The alerts API lets you set price and technical alerts on NSE/BSE symbols. When an alert fires, the server can POST a webhook payload to a URL you specify — no polling required. Alerts persist across server restarts (saved to `~/.trading_platform/alerts.json`).

---

## POST /skills/alerts/add

Create a new alert. Three alert types are supported:

- **Price alert** — fires when the last traded price crosses a threshold
- **Technical alert** — fires when an indicator (RSI, MACD, ADX, ATR, SCORE) crosses a level
- **Conditional alert** — fires when ALL conditions in a list are met simultaneously (AND logic)

### Price alert

```json
{
  "symbol": "RELIANCE",
  "condition": "ABOVE",
  "threshold": 2800
}
```

### Price alert with webhook

```json
{
  "symbol": "RELIANCE",
  "condition": "ABOVE",
  "threshold": 2800,
  "webhook_url": "https://my-agent.example.com/on-alert"
}
```

### Technical alert

```json
{
  "symbol": "INFY",
  "indicator": "RSI",
  "condition": "ABOVE",
  "threshold": 70
}
```

### Conditional alert (AND logic)

```json
{
  "symbol": "RELIANCE",
  "conditions": [
    { "condition_type": "PRICE",     "condition": "ABOVE", "threshold": 2800 },
    { "condition_type": "TECHNICAL", "condition": "ABOVE", "threshold": 60, "indicator": "RSI" }
  ]
}
```

### Parameters

<ParamField body="symbol" type="string" required>
  NSE/BSE symbol (e.g., `RELIANCE`, `NIFTY`).
</ParamField>

<ParamField body="exchange" type="string" default="NSE">
  Exchange: `NSE` or `BSE`. Defaults to `NSE`.
</ParamField>

<ParamField body="condition" type="string">
  `ABOVE`, `BELOW`, or `CROSSES`. Required for price and technical alerts.
</ParamField>

<ParamField body="threshold" type="number">
  Price or indicator level to trigger at. Required for price and technical alerts.
</ParamField>

<ParamField body="indicator" type="string">
  Technical indicator for technical alerts. Supported values: `RSI`, `MACD`, `ADX`, `ATR`, `SCORE`. Omit for a plain price alert.
</ParamField>

<ParamField body="conditions" type="array">
  List of condition objects for conditional (AND-logic) alerts. Each object should include `condition_type` (`PRICE` or `TECHNICAL`), `condition`, `threshold`, and optionally `indicator`.
</ParamField>

<ParamField body="webhook_url" type="string">
  URL to POST when the alert fires. Optional for all alert types. When omitted, use `/skills/alerts/check` to poll for triggered alerts.
</ParamField>

### Example

<CodeGroup>

```bash curl
curl -X POST http://localhost:8765/skills/alerts/add \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "RELIANCE",
    "condition": "ABOVE",
    "threshold": 2800,
    "webhook_url": "https://my-agent.example.com/on-alert"
  }'
```

```python python
import requests

resp = requests.post(
    "http://localhost:8765/skills/alerts/add",
    json={
        "symbol": "RELIANCE",
        "condition": "ABOVE",
        "threshold": 2800,
        "webhook_url": "https://my-agent.example.com/on-alert",
    },
)
print(resp.json())
```

</CodeGroup>

### Response

```json
{
  "status": "ok",
  "data": {
    "id": "abc12345",
    "alert_type": "PRICE",
    "symbol": "RELIANCE",
    "condition": "ABOVE",
    "threshold": 2800.0,
    "description": "RELIANCE price ABOVE ₹2,800.00",
    "created_at": "2026-04-04T09:15:00",
    "webhook_url": "https://my-agent.example.com/on-alert"
  }
}
```

### Webhook payload on trigger

When an alert fires, the server sends this JSON body via POST to your `webhook_url`:

```json
{
  "event": "alert_triggered",
  "alert_id": "abc12345",
  "symbol": "RELIANCE",
  "description": "RELIANCE price ABOVE ₹2,800.00",
  "triggered_at": "2026-04-03T11:00:00",
  "ltp": 2815.5
}
```

Your webhook endpoint should respond with any `2xx` status to acknowledge receipt.

## POST /skills/alerts/list

Return all active (not yet triggered) alerts.

### Request body

```json
{}
```

No input fields required.

### Example

```bash
curl -X POST http://localhost:8765/skills/alerts/list \
  -H "Content-Type: application/json" \
  -d '{}'
```

### Response

```json
{
  "status": "ok",
  "data": [
    {
      "id": "abc12345",
      "alert_type": "PRICE",
      "symbol": "RELIANCE",
      "condition": "ABOVE",
      "threshold": 2800.0,
      "description": "RELIANCE price ABOVE ₹2,800.00",
      "created_at": "2026-04-04T09:15:00",
      "webhook_url": "https://my-agent.example.com/on-alert"
    },
    {
      "id": "def67890",

      "description": "INFY RSI ABOVE 70.00",
      "created_at": "2026-04-04T09:20:00",
      "webhook_url": null
    }
  ]
}
```

## POST /skills/alerts/remove

Remove an alert by its ID. The ID is returned when the alert is created and is also visible in the `/skills/alerts/list` response.

### Request body

```json
{ "alert_id": "abc12345" }
```

<ParamField body="alert_id" type="string" required>
  Alert ID to remove.
</ParamField>

### Example

<CodeGroup>

```bash curl
curl -X POST http://localhost:8765/skills/alerts/remove \
  -H "Content-Type: application/json" \
  -d '{"alert_id": "abc12345"}'
```

```python python
import requests

resp = requests.post(
    "http://localhost:8765/skills/alerts/remove",
    json={"alert_id": "abc12345"},
)
print(resp.json())
```

</CodeGroup>

### Response

```json
{
  "status": "ok",
  "data": { "alert_id": "abc12345", "removed": true }
}
```

Returns a `404` error if the alert ID does not exist.

## POST /skills/alerts/check

Manually evaluate all active alerts against current prices right now. Returns any alerts that fired during this call, and the count of alerts still being watched.

Use this endpoint if you prefer polling over webhooks. For push-based agents, register a `webhook_url` when creating alerts instead.

### Request body

```json
{}
```

No input fields required.

### Example

```bash
curl -X POST http://localhost:8765/skills/alerts/check \
  -H "Content-Type: application/json" \
  -d '{}'
```

### Response

```json
{
  "status": "ok",
  "data": {
    "triggered": [
      {
        "id": "abc12345",
        "symbol": "RELIANCE",
        "description": "RELIANCE price ABOVE ₹2,800.00",
        "triggered_at": "2026-04-04T11:00:00",
        "ltp": 2815.5
      }
    ],
    "active_remaining": 1
  }
}
```

<ResponseField name="triggered" type="array">
  Alerts that fired during this check. Empty array if none triggered.
</ResponseField>

<ResponseField name="active_remaining" type="number">
  Number of alerts still active and being watched after this check.
</ResponseField>
